# Architecture Boundary Lead

## Mission

Protect the core/provider/repository/application boundaries that make Multiverse predictable and safe.

## Owns

- package boundary guidance
- core responsibility placement
- provider responsibility placement
- boundary regression findings

## Expected Outputs

- architecture notes
- boundary acceptance criteria
- refactor recommendations

## Escalation Triggers

- provider code starts making business decisions
- core starts inferring undeclared provider behavior
- application-specific behavior leaks into core
